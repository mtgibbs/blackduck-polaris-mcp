## Codebase Patterns

- **3-layer architecture**: MCP Tool → Service Layer → API Layer
  - Tools in src/mcp/tools/ define schemas and call services
  - Services in src/services/ contain business logic
  - API layer in src/api/ handles HTTP calls with vendor media types
- **Service imports**: Always import from services/index.ts, never from api/ directly
- **ProjectMapping structure**: Has `id`, `projectId`, `configurationId`, `btsProjectKey`, `btsIssueType` (default issue type)
- **Error patterns**: Return `{error: string}` for user-facing error messages with actionable guidance
