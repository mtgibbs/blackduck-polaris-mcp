## Codebase Patterns
- Bug tracking project mappings use `getConfigProjectMappings({configurationId})` from services layer
- `ProjectMapping.btsIssueType` is a string, not an object (contains the type ID directly)
- Service layer functions should be imported from `../services/index.ts`
- TypeScript return types for helpers use union types for success/error: `Promise<{success} | {error}>`
