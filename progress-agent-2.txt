## Codebase Patterns

- Filter validation is advisory-only: warns but does not block unknown keys (API may add new keys)
- RSQL filter key extraction uses simple regex, not a full parser: /([a-zA-Z0-9:._-]+)\s*(?:==|!=|=in=|=out=|=like=)/g
- Portfolio tools use bare filter keys (name, description, etc.)
- Findings tools use namespaced filter keys (occurrence:, context:, special:, triage:, type:)
- Tests and Bug Tracking tools use bare filter keys
